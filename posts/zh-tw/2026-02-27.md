## 什麼是元提示詞？

> 元提示詞（Meta-prompting）簡單來說就是**「用來製作提示詞的提示詞」，或者「給AI高階指令來控制其行為和思考流程」**。與通常的提示詞要求AI提供「答案」不同，元提示詞是給AI提供「應該如何思考、如何構建答案」的結構和規則。

雖然AI會如上所述進行解釋，但

**說白了，就是讓AI來寫提示詞！**

舉例來說，

```
請生成用於資料製作的提示詞。
```

像這樣下指令的話，

```
你是一位優秀的【角色（例：經營顧問 / 行銷人員 / 銷售人員）】。
根據以下要求，製作【資料類型（例：新事業企劃書 / 銷售用簡報資料結構 / 業務改善提案書）】。

# 目的
・【透過此資料要達成的目標（例：獲得上司的預算批准 / 向客戶傳達新服務的魅力）】

# 目標對象（讀者·聽眾）
・【資料面向的對象（例：IT素養不高的經營層 / 既有客戶的負責人）】

# 應包含的要素·前提條件
・【要素1（例：現狀課題：〇〇的成本較高）】
・【要素2（例：解決方案：導入新系統實現自動化）】
・【要素3（例：預期效果：成本削減20%）】

# 語調與風格
・【期望的文體（例：邏輯清晰、說服力強的語調 / 避免專業用語的易理解表現）】

# 輸出格式
・【期望的格式（例：請以全5頁的幻燈片結構方案形式輸出，每頁列出標題和應傳達的訊息的項目符號）】
```

這樣AI就會為你生成提示詞。

有[研究](https://arxiv.org/abs/2503.16789)表明，讓LLM自己來寫提示詞**效果更好**，敬請活用！
